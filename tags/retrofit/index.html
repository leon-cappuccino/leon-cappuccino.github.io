Android网络技术

2022年3月11日
| Android基础知识 Android网络
Android网络技术 使用网络需要添加网络权限 <uses-permission android:name="android.permission.INTERNET"/> Retrofit 添加依赖: implementation 'com.squareup.retrofit2:retrofit:2.6.1' //导入Retrofit、OkHttp和Okio库 implementation 'com.squareup.retrofit2:converter-gson:2.6.1' //Retrofit的转换库:GSON库 Retrofit是基于OkHttp开发的，OkHttp侧重于底层，Retrofit更侧重于顶层封装。 retrofit共有22个注解，其中22个注解又可以分为请求方法类，标记类和参数类。 1.Retrofit的使用方法 例如在一个天气App的开发中，我们想要服务器的API接口 https://api.caiyunapp.com/v2/place?query=北京&token={token}&lang=zh_CN 返回的接口数据如下 {"status":"ok","query":"北京", "places":[ {"name":"北京市","location":{"lat":39.9041999,"lng":116.4073963}, "formatted_address":"中国北京市"}, {"name":"北京西站","location":{"lat":39.89491,"lng":116.322056}, "formatted_address":"中国 北京市 丰台区 莲花池东路118号"}, {"name":"北京南站","location":{"lat":39.865195,"lng":116.378545}, "formatted_address":"中国 北京市 丰台区 永外大街车站路12号"} ]} （1）定义数据接收模型 首先根据接口返回的数据定义数据模型，新建PlaceResponse文件 class PlaceResponse(val status: String, val places: List<Place>) class Place(val name: String, val location: Location, @SerializedName("formatted_address") val address: String) class Location(val lng: String, val lat: String) @SerializedName注解可以将json中的属性名转为我们自己自定义的属性名，即将formatted_address转换为address。 （2）定义访问API的retrofit接口 我们需要定义一个API接口将数据接收模型 对于API接口 https://api.caiyunapp.com/v2/place?query=北京&token={token}&lang=zh_CN 动态变化指定的参数为query，对于这样的API我们使用@Query用于 @GET 方法的查询参数。……
